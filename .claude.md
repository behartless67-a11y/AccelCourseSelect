# Accelerated Course Selection System

## Project Overview
Web application for Frank Batten School's accelerated program course selection system. Replaces Salesforce with a real-time, live-updating platform.

## System Purpose
- Students select 1st, 2nd, and 3rd choice courses
- Real-time seat availability (WebSocket updates)
- Admin interface for course management and CSV uploads
- Optimization algorithm to assign students based on preferences

## Tech Stack
- **Frontend**: React with WebSocket client
- **Backend**: Node.js + Express + Socket.io
- **Database**: PostgreSQL
- **Assignment Algorithm**: Python script for optimization
- **Real-time**: Socket.io for instant seat updates

## Key Features
1. **Student Portal**: Browse courses, see live availability, submit ranked preferences
2. **Admin Dashboard**: Upload courses (CSV), manage capacities, run assignment algorithm
3. **Real-time Updates**: Instant seat count updates across all connected users
4. **Optimization**: Weighted assignment algorithm maximizing student satisfaction

## Course Structure
- Courses organized by subject (ECON, RMDA, POLICY, etc.)
- Each course has Sections (lectures) and Discussions
- Each section/discussion has capacity limits
- Students rank their top 3 choices

## Assignment Rules
- Weight preferences: 1st choice = 3 points, 2nd = 2 points, 3rd = 1 point
- Maximize total satisfaction while respecting capacities
- Handle oversubscribed courses fairly
